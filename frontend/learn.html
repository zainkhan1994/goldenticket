Storing the Data
Where does all the information live before the AI sees it?
📚

🌟 Think of it like this

Imagine a magic library in the cloud where every book (your health data) is stored in a super-smart way. Instead of reading each book from page 1, you can jump straight to the exact chapter you need — saving tons of time!

📦 Parquet reads only the columns you need (much faster than CSV!)

Old way (CSV) 🐢

NameSleepEnergyFocus

Reads ALL columns 😩

Parquet 🚀

Sleep ← only this!

Reads ONE column ✅

🏷️ Format
Apache Parquet — stores data in columns, not rows
🏠 Where
Google Cloud Storage — like Google Drive but for huge data files
⚡ Speed boost
Reading 1 of 50 columns is ~50× faster with Parquet
💾 Space saving
Parquet also compresses the data — like zipping a file!
🔑 Key idea: The smarter you store data, the faster you can read it later. Parquet is like organizing your toy box so you can find any toy instantly!

Superfast GPU Processing
How do we crunch millions of numbers in milliseconds?
🧮

🌟 Think of it like this

Your regular computer processor (CPU) is like 1 super-genius mathematician solving problems one at a time. A GPU is like having thousands of helpers all solving different parts of the problem at the same exact moment — incredibly fast!

🔵 Each little square = one GPU core working right now!

CPU: 8 cores GPU: 9,000+ cores ⚡
🛠️ Tool used
cuDF — pandas but on the GPU (looks the same, runs 100× faster)
🧱 What it does
Reads your Parquet data and cleans + analyzes millions of rows
⏱️ Speed example
A 45-second CPU calculation becomes 0.4 seconds on the GPU!
🛡️ Safety feature
UVM (Unified Virtual Memory) means it won't crash if the data is too big

📊 Data flowing from storage into the GPU right now!

Sleep data
Energy data
Focus data
🔑 Key idea: GPUs were invented for video games (to draw millions of pixels at once), but scientists realized they are also perfect for AI math! 🎮 → 🤖

The Kubernetes Kitchen
How do we keep the AI running safely in the cloud?
🍳

🌟 Think of it like this

Kubernetes is like a restaurant kitchen manager. The manager knows exactly which chef (computer) is busy and which is free. If a chef gets sick (a computer breaks), the manager immediately sends another chef to take over — so food (your AI responses) keeps coming without any interruption!

🖥️ Inside our Google Cloud cluster — everything running right now

kube-dns
metrics-server
driver-installer
🤖 NIM AI Pod (GPU!)
node-agent
log-collector
🌍 Where
Google Cloud (GKE) — Google manages the cluster for us
🔧 Set up with
Just one shell script — the computer does the rest!
💰 Smart savings
The GPU node turns off automatically when not used (~$0.70/hr saved)
📦 Helm
Like a vending machine for apps — pick what you want and it installs!
🔑 Key idea: Kubernetes was invented by Google engineers who needed to run thousands of websites at once — now anyone can use it! 🌐

The AI Brain (NVIDIA NIM)
Where does the smart, helpful advice actually come from?
📖

🌟 Think of it like this

The AI model (Llama 3) is like someone who has read billions of books and articles. NVIDIA NIM is like a special racing car that lets this genius run as fast as physically possible on a GPU — instead of slowly walking (regular computer), it zooms! 🏎️

Llama 3 AI — optimized by TensorRT for max speed

Talking to the AI (The Browser)
How does your question travel to the AI and come back?
📮

🌟 Think of it like this

The browser is like a super-fast post office. You write a letter (your question), it gets delivered to the AI brain through a secret tunnel (port-forward), the AI writes back, and the post office shows you the reply — all in under 2 seconds! 📬

✉️ Watch your message travel to the AI and come back!

👤
You type: "I haven't slept well in a week…"
⬇️ HTTP POST (fancy word for "send message")
🔀
Secret tunnel (kubectl port-forward) passes it along
⬇️ to the GPU cluster
🧠
AI thinks for ~1–2 seconds on the GPU, writes a response
⬆️ JSON response (a tidy package of text)
✅
Browser shows the reply in a chat bubble — done!
📄 Built with
Just one HTML file — no apps to install, open in any browser
💾 Memory
The browser remembers your whole conversation so the AI knows context
🚦 Status dot
A green/red dot shows if the AI is online or offline instantly
♿ Accessible
Works with screen readers so everyone can use it
🔑 Key idea: The browser talks to the AI using the exact same format as ChatGPT! This is called an "OpenAI-compatible API" — like a universal language that all AI tools understand. 🌐

📋 Quick Start Commands

1

Clone the project

git clone https://github.com/zainkhan1994/goldenticket.git
2

Create the GPU cluster

bash scripts/setup_gke.sh
3

Deploy the AI model

export NGC_API_KEY="your-key" && bash scripts/deploy_nim.sh
4

Connect your browser to the AI

kubectl port-forward service/nim-llm 8000:8000
